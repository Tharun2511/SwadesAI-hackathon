# Verbis — AI Meeting Transcriber

Verbis is an AI-powered meeting transcription app that records conversations directly in the browser, automatically identifies multiple speakers, and generates an AI summary. Transcripts are saved per user by email and accessible across devices.

## Features

- **Browser recording** — one-click record via MediaRecorder API, no installs needed
- **Speaker diarization** — multiple speakers automatically identified as User1, User2, etc. with distinct colors
- **Accurate transcription** — powered by Deepgram nova-2 with smart formatting and punctuation
- **Timestamps** — each speaker segment shows its start time
- **AI summary** — 3-bullet TL;DR generated by Groq (LLaMA 3.1) immediately after transcription
- **Speaker renaming** — rename User1 → "Alice" inline; changes persist to DB across devices
- **Transcription naming** — give each recording a custom title for easy reference
- **Cross-device history** — transcriptions saved per email, viewable from any device
- **Export** — copy transcript to clipboard or download as `.txt` or `.md`
- **Delete** — remove individual transcriptions from history
- **Microphone error guidance** — clear instructions if browser blocks mic access

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/transcribe` | Receives audio (multipart), transcribes via Deepgram, generates summary via Groq, saves to DB |
| `GET` | `/api/transcriptions` | Returns last 20 transcriptions for a given `?email=` |
| `PATCH` | `/api/transcriptions/:id` | Update title and/or rename speakers |
| `POST` | `/api/transcriptions/:id/summary` | Generate AI summary on demand for older records |
| `DELETE` | `/api/transcriptions/:id` | Delete a transcription |
| `POST` | `/api/chat` | Groq streaming chat (LLaMA 3.1) |

### POST /api/transcribe

**Request:** `multipart/form-data` — fields `audio` (WebM blob) and `email` (string)

**Response:**
```json
{
  "id": "uuid",
  "duration": 42.3,
  "summary": "• Key topic one\n• Decision made\n• Action item",
  "segments": [
    { "speaker": "User1", "text": "Hello, how are you?", "start": 0.1, "end": 2.4 },
    { "speaker": "User2", "text": "I'm doing great, thanks.", "start": 2.8, "end": 5.1 }
  ]
}
```

### PATCH /api/transcriptions/:id

**Request body** (all fields optional):
```json
{
  "title": "Q2 Planning Call",
  "speakerMap": { "User1": "Alice", "User2": "Bob" }
}
```

## Local Setup

### 1. Clone and install

```bash
git clone https://github.com/Tharun2511/Swades-AI-Hackathon.git
cd Swades-AI-Hackathon
npm install
```

### 2. Environment variables

Copy `.env.example` to `.env` and fill in your keys:

```bash
cp .env.example .env
```

```env
DATABSE_URL=postgresql://<user>:<password>@<host>/neondb?sslmode=require&channel_binding=require
DEEPGRAM_API_KEY=your_deepgram_api_key
GROQ_API_KEY=your_groq_api_key
```

> **Note:** The variable is `DATABSE_URL` (typo preserved from original setup) — use exactly this spelling.

Get your keys:
- **Neon** — [neon.tech](https://neon.tech) — free tier, serverless PostgreSQL
- **Deepgram** — [deepgram.com](https://deepgram.com) — free $200 credit on signup
- **Groq** — [console.groq.com](https://console.groq.com) — free tier

### 3. Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). The database schema is auto-created on first run via TypeORM `synchronize: true` — no migrations needed.

### 4. Browser microphone permission

On first use the browser will prompt for mic access — click **Allow**. If you previously blocked it:

> Chrome: click the 🔒 lock icon → Site settings → Microphone → set to **Ask** or **Allow** → refresh

## How It Works

1. User enters their email — transcriptions are scoped to that email and synced across devices
2. Click **Start Recording** → browser requests mic via `getUserMedia`
3. `MediaRecorder` captures audio in 250ms chunks (WebM/Opus)
4. On **Stop**, chunks are assembled into a Blob and sent as `multipart/form-data` to `POST /api/transcribe`
5. Deepgram nova-2 transcribes the audio with `diarize: true` — returns word-level data with speaker labels
6. Words are grouped into consecutive speaker segments and mapped to User1, User2, etc.
7. Groq (LLaMA 3.1 8B) generates a 3-bullet summary from the transcript text
8. Segments, duration, and summary are saved to Neon PostgreSQL via TypeORM
9. The transcript, summary, and updated history are shown instantly on the frontend
10. Users can rename speakers, name the recording, copy/download the transcript — all changes persist to DB
